1. Project Introduction

Project title: ComLab V3 Emergency Egress Agent Simulation.

The system models an emergency evacuation inside ComLab V3 at UMTC Visayan Campus, including students, the professor, student assistants, custodians, lockers, data racks, doors, stairs, smoke, and congestion.

The study matters because a computer laboratory has dense furniture, narrow paths, electronics, and human delay behaviors that can slow evacuation during a fire event.

2. Problem Definition

The current layout places service/locker activity near the exit path, creating cross-traffic, door jams, and high congestion during evacuation.

The simulation analyzes whether moving service-zone pressure away from the exit path improves evacuation time, queue behavior, and incident frequency.

3. Objectives

  • Minimize total evacuation and average waiting time.
  • Reduce queue length and congestion heat near bottlenecks.
  • Improve use of exit-door resources and staff guidance.
  • Compare current and modified layout scenarios.

4. System Model and Design

Entities
36 students, 1 professor, 2 student assistants, 2 custodians.
Events
Pack-up delay, locker retrieval, peer waiting, movement, trip/faint, door jam, extinguisher retrieval, exit.
Resources
Two exits, hallway/stair paths, service-bay passage, lockers, fire extinguishers, staff aides.
Queues
Center-aisle and exit-approach queues sampled over time for average queue length.
State variables
Time, position, phase, target, wait time, heatmap counts, evacuation rate, trips, door collisions.

Process Flow

  1. Incident trigger: Fire and smoke initiate, and agents enter behavioral reaction delays.
  2. Pre-evacuation: Agents pack up, wait for peers, or (if staff) retrieve fire extinguishers.
  3. Service detour: Locker-bound agents detour to the service zone to retrieve belongings.
  4. Movement: Agents pathfind to assigned exits, slowed by smoke and crowd density.
  5. Evacuation: Agents navigate bottlenecks, trips, and door jams to successfully exit.

5. Assumptions and Input Data

Actual observed timing data is unavailable, so the model uses deterministic seeded assumptions for repeatable comparison.

  • Activation/arrival into motion: 2-12 seconds by behavior type.
  • Service delays: locker 3-15 seconds, extinguisher 4 seconds, trip/faint recovery 3-18 seconds.
  • Door jams last 3-5 seconds; smoke and crowd density reduce movement speed.
  • Limitations: grid movement, fixed population, fixed floor plan, no continuous physics.

6. Implementation Demo

The live canvas demonstrates time progression, event processing, queue/heatmap accumulation, and output generation from the Python engine.

Controls expose scenario variables: layout mode, panic behavior, fire origin, speed, stepping, reset, heatmap, and side-by-side comparison.

7. Results and Analysis

The comparison table reports total processing time, average wait, average queue length, throughput, exit utilization, trips, door collisions, and congestion heat.

At minimum, it compares the current layout against the modified layout under the same panic and fire-origin settings.

8. Conclusion and Recommendations

Expected findings: the modified layout shortens evacuation, lowers bottleneck queues, and reduces door-related incidents by clearing the exit path.

Recommended improvements are to keep locker/service activity away from exits, preserve clear aisle access, and assign staff to guide each exit.
